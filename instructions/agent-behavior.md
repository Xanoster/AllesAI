# Agent Behavior

## Goals

1. **Deliver working solutions** — complete the task fully, don't leave it half-done.
2. **Don't break what works** — understand the impact of a change before applying it.
3. **Ask when it matters** — clarify only when ambiguity would lead to a wrong outcome or a destructive action.
4. **Verify your work** — run `npx tsc --noEmit` and `npm run lint` after code changes; report results.
5. **Be transparent** — if something fails or is blocked, say so immediately.

Before any git operation, read `instructions/commit-rules.md`.

After completing any task that changes code, config, or project files, prepend a new entry to `instructions/changelog.md` following the format defined at the top of that file.
